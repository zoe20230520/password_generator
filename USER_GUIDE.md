# 密码管理系统 - 使用指南

## 系统概述

这是一个安全、便捷的密码管理系统，支持用户注册登录、密码存储、密码生成、分类管理等功能。

## 系统要求

- 操作系统：Windows 7/8/10/11 或更高版本
- Python 3.7 或更高版本（**必须安装**）
- 现代浏览器（Chrome、Firefox、Edge、Safari 等）
- 至少 100MB 可用磁盘空间

### ⚠️ 重要提示：数据库软件

**不需要单独安装数据库软件！**

系统使用 SQLite 数据库，它是一个轻量级的嵌入式数据库：
- ✓ **无需安装**：Python 内置支持，无需额外下载
- ✓ **零配置**：首次运行会自动创建 `database.db` 文件
- ✓ **即开即用**：启动后立即可用
- ✓ **易于备份**：只需复制 `database.db` 文件即可备份数据

## 快速启动

### 方式一：使用启动脚本（推荐）

1. 双击运行 `start.bat` 文件
2. 等待依赖包安装完成
3. 浏览器自动打开访问地址：`http://localhost:5000`

### 方式二：手动启动

1. 打开命令行（CMD）或 PowerShell
2. 进入项目目录：
   ```bash
   cd password-manager
   ```
3. 安装依赖（首次运行需要）：
   ```bash
   pip install -r backend\requirements.txt
   ```
4. 启动服务：
   ```bash
   python backend\app.py
   ```
5. 浏览器访问：`http://localhost:5000`

### 如果 start.bat 没有反应

请查看 **[TROUBLESHOOTING.md](TROUBLESHOOTING.md)** 获取详细解决方案

快速排查步骤：
1. 打开命令行（CMD）
2. 进入项目目录：`cd password-manager`
3. 手动运行：`start.bat`
4. 查看错误信息

## 用户使用流程

### 1. 用户注册

首次使用需要注册账号：

1. 访问系统首页，系统会自动跳转到登录页面
2. 点击"立即注册"按钮
3. 填写注册信息：
   - **用户名**：3-20个字符
   - **邮箱**：有效的邮箱地址
   - **密码**：至少6个字符
4. 勾选"我已阅读并同意服务条款"
5. 点击"注册"按钮完成注册

### 2. 用户登录

1. 在登录页面输入用户名和密码
2. 点击"登录"按钮
3. 登录成功后进入密码管理系统主页

### 3. 添加密码记录

1. 点击页面右上角的"添加密码"按钮
2. 填写密码信息：
   - **网站名称**：必填，例如：Google、Facebook
   - **网站地址**：可选，例如：https://www.google.com
   - **用户名/邮箱**：必填
   - **密码**：必填，可以手动输入或使用生成器
   - **分类**：可选，例如：社交媒体、工作、财务等
   - **备注**：可选，添加额外说明
3. 可以上传网站图标（可选）
4. 点击"保存"按钮

### 4. 生成强密码

使用密码生成器创建安全密码：

1. 在添加/编辑密码时，点击密码输入框旁的"生成"按钮
2. 配置密码选项：
   - **长度**：6-32位，推荐12位以上
   - **包含字母**：大小写字母
   - **包含数字**：0-9
   - **包含符号**：!@#$%^&*()_+-=[]{}|;:,.<>?
3. 实时查看密码强度评估
4. 点击"使用此密码"应用生成的密码

### 5. 查看密码记录

1. 主页显示所有保存的密码记录
2. 默认按最近修改时间排序
3. 使用搜索框快速查找：
   - 支持搜索网站名称、用户名、备注
4. 使用分类筛选器按类别过滤

### 6. 复制密码

1. 点击密码记录右侧的"复制"按钮
2. 密码会自动复制到剪贴板
3. 提示消息确认复制成功

### 7. 查看完整密码

1. 点击密码记录中的"查看密码"按钮（眼睛图标）
2. 密码会以明文显示
3. 再次点击可隐藏密码

### 8. 编辑密码记录

1. 点击密码记录右侧的"编辑"按钮
2. 修改需要更新的字段
3. 点击"保存"按钮确认修改

### 9. 删除密码记录

1. 点击密码记录右侧的"删除"按钮
2. 在确认对话框中点击"确认删除"
3. 密码记录将被永久删除

### 10. 分类管理

1. 系统会自动根据已使用的分类生成分类列表
2. 使用分类筛选器查看特定类别的密码
3. 在添加/编辑记录时可以创建新分类

### 11. 退出登录

1. 点击页面右上角的用户名
2. 点击"退出登录"按钮
3. 系统会清除登录状态并返回登录页面

## 数据安全说明

### 密码加密

- 所有密码在数据库中都经过加密存储
- 使用行业标准的加密算法
- 无法直接查看明文密码

### 用户认证

- 基于 JWT Token 的身份验证
- Token 有效期为 7 天
- 自动保护用户数据隔离

### 安全建议

1. **使用强密码**：密码长度至少 12 位，包含大小写字母、数字和符号
2. **定期更换密码**：建议每 3-6 个月更换重要账户的密码
3. **不要重复使用密码**：为不同网站使用不同的密码
4. **及时保存**：生成的密码要及时保存到系统中
5. **定期备份**：定期备份 `database.db` 文件

### 本地部署安全

如果要在生产环境使用，建议：

1. 修改 `backend/config.py` 中的 `SECRET_KEY` 为随机密钥
2. 使用 HTTPS 而非 HTTP
3. 设置防火墙规则，限制访问端口
4. 定期更新依赖包：`pip install --upgrade -r backend/requirements.txt`

## 常见问题

### Q: 忘记登录密码怎么办？

A: 由于安全考虑，系统不支持密码找回功能。建议您记住主密码或使用密码管理器记住。

### Q: 需要安装数据库软件吗？

A: **不需要！** 系统使用 SQLite 数据库，它是 Python 内置的，首次运行会自动创建 `database.db` 文件。

### Q: start.bat 没有反应怎么办？

A: 请查看 **[TROUBLESHOOTING.md](TROUBLESHOOTING.md)** 获取详细的解决方案。

最常见原因：
- Python 未安装或未添加到 PATH
- 可以在命令行中手动运行 `start.bat` 查看错误信息

### Q: 依赖安装失败怎么办？

A:
1. 更新 pip：`python -m pip install --upgrade pip`
2. 使用国内镜像源：`pip install -r backend\requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple`
3. 详见 [TROUBLESHOOTING.md](TROUBLESHOOTING.md)

### Q: 可以在多个设备上使用吗？

A: 当前版本仅支持本地部署。如需多设备使用，建议：
- 将整个项目文件夹复制到其他设备
- 复制 `database.db` 文件到新设备

### Q: 如何备份数据？

A: 备份 `database.db` 文件即可，该文件包含所有用户和密码数据。

### Q: 系统支持多少人使用？

A: 理论上支持无限用户，但受限于服务器性能。

### Q: 浏览器兼容性如何？

A: 支持所有现代浏览器，包括：
- Chrome 60+
- Firefox 55+
- Edge 79+
- Safari 11+

### Q: 如何停止服务？

A: 在命令行窗口按 `Ctrl+C` 停止服务。

### Q: 端口被占用怎么办？

A: 修改 `backend/app.py` 文件最后一行：
```python
app.run(debug=True, host='0.0.0.0', port=5000)
```
将 `port=5000` 改为其他端口号，如 `port=5001`。

### Q: 遇到其他问题怎么办？

A: 请查看 **[TROUBLESHOOTING.md](TROUBLESHOOTING.md)** 获取详细的故障排除指南。
